¿QUE ES WEBASSEMBLY?
WebAssembly

WebAssembly, abreviado wasm, es un formato de código binario portable, para la ejecución íntegra en navegador de scripts de lado del cliente. Se trata de un lenguaje de bajo nivel, diseñado inicialmente como formato destino en la compilación desde C y C++.

Eficiente y Rapido

La máquina de pila Wasm está diseñada para codificarse en un formato binario eficiente en tamaño y tiempo de carga . WebAssembly tiene como objetivo ejecutarse a velocidad nativa aprovechando las capacidades de hardware comunes disponibles en una amplia gama de plataformas.

Portabilidad

El formato binario de WebAssembly está diseñado para ser ejecutable de manera eficiente en una variedad de sistemas operativos y arquitecturas de conjunto de instrucciones, en la Web y fuera de la Web .

API

WebAssembly no especifica ninguna API ni llamadas al sistema, solo un mecanismo de importación en el que el entorno del host define el conjunto de importaciones disponibles. En un entorno Web , se accede a la funcionalidad a través de las API Web definidas por la Plataforma Web . Los entornos no web pueden optar por implementar API web estándar, API no web estándar (por ejemplo, POSIX) o inventar las suyas propias.

Nivel de origen

La portabilidad en el nivel de C/C++ se puede lograr mediante la programación de una API estándar (por ejemplo, POSIX) y confiando en el compilador y/o las bibliotecas para asignar la interfaz estándar a las importaciones disponibles del entorno host en tiempo de compilación (a través de #ifdef) o tiempo de ejecución (a través de detección de características y carga / vinculación dinámica ).

SEGURO

WebAssembly describe un entorno de ejecución en espacio aislado y seguro para la memoria que incluso puede implementarse dentro de las máquinas virtuales de JavaScript existentes. Cuando se incrusta en la web , WebAssembly aplicará las mismas políticas de seguridad de origen y permisos del navegador.

Abierto y depurable

WebAssembly está diseñado para imprimirse de forma bonita en un formato de texto para depurar, probar, experimentar, optimizar, aprender, enseñar y escribir programas a mano. El formato textual se utilizará cuando se visualice la fuente de los módulos Wasm en la web.

Parte de la plataforma web abierta

WebAssembly está diseñado para mantener la naturaleza de la web sin versiones, con funciones probadas y compatible con versiones anteriores . Los módulos de WebAssembly podrán entrar y salir del contexto de JavaScript y acceder a la funcionalidad del navegador a través de las mismas API web accesibles desde JavaScript. WebAssembly también admite incrustaciones no web .

Cómo funciona el motor JS

Para compilar el código JS, hay algunas cosas que el motor debe hacer:

  1. 1. El analizador : el motor debe pasar ese código a través de un analizador primero. El analizador recorre el código línea por línea y verifica la sintaxis válida, así como los tipos de código. Si todo es válido, el analizador crea una estructura de datos de árbol denominada árbol de sintaxis abstracta (AST).
  2. 2. AST a representación intermedia (IR) : luego, el intérprete del motor toma el AST y lo convierte en Bytecode, que es una representación intermedia del código (un IR es una abstracción del código de máquina).
  3. 3. Compilación del IR al código de la máquina : solo entonces el compilador del motor toma el código de bytes y lo convierte en un código que una máquina puede ejecutar en su procesador.

¿Dónde está Web Assembly hoy?

En 2019, investigadores de Braunschweig en Alemania analizaron el millón de sitios web principales de Alexa y su uso de Web Assembly. Querían ver para qué se usa WASM hoy. Analizaron 947.407 sitios web y 3.465.320 páginas. Descubrieron que 1639 sitios web están cargando 1950 módulos de WASM.
Los dos usos principales de WASM fueron:

  1. 1. Criptominería : el 32% de los sitios web utilizaron WASM para la minería de criptomonedas (principalmente con fines maliciosos en sitios web pirateados). La razón por la que usaron WASM es que el aumento de velocidad también significa que el hardware de la computadora se puede explotar de manera más efectiva para los ataques de cryptojacking.
  2. 2. Juegos : el 29,3% de los sitios web utilizaron WASM para juegos. Un ejemplo es el popular juego Doom 3. Sin embargo, hay más usos para Web Assembly que solo Gaming y Crypto Mining. De hecho, Figma, una popular herramienta de diseño web, informó que triplicaron su velocidad de carga con WASM.
